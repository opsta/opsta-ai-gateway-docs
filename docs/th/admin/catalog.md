# แคตตาล็อก Agent และเครื่องมือ (Agent & Tool catalog)

**แคตตาล็อก Agent และเครื่องมือ (Agent & Tool catalog)** คือหน้ารายงานการกำกับดูแลภาพรวมของทุกส่วนงานที่สามารถปฏิบัติการได้ในองค์กรของคุณ ไม่ว่าจะเป็น **AI agent** หรือ**เครื่องมือภายใต้การควบคุม**ในทุก ๆ โปรเจกต์ พร้อมทั้งระบุข้อมูล**ผู้รับผิดชอบ (owner)** และ**ระดับความเสี่ยง (risk tier)** เมื่อมีการสร้าง agent และเครื่องมือเพิ่มขึ้นเป็นจำนวนมาก หน้านี้จะเป็นศูนย์กลางเพียงจุดเดียวในการตอบคำถามว่า "มีระบบใดอยู่บ้าง ใครเป็นเจ้าของ และระบบนั้นมีความเสี่ยงระดับใด"

::: info ผู้ที่มีสิทธิ์ในการดำเนินการนี้
**Org admin** (สิทธิ์เฉพาะในองค์กรของตนเอง) และ **Platform admin** โดยเข้าใช้งานผ่านหัวข้อ **Agent & Tool catalog** ในเมนูด้านซ้ายของผู้ดูแลระบบ
:::

## ข้อมูลที่แสดงบนหน้าจอ

- **Agents:** รายชื่อ [AI agent](/th/admin/agents) ที่ลงทะเบียนไว้ทั้งหมด ได้แก่ ชื่อ โปรเจกต์ กลุ่ม ผู้ที่ปฏิบัติงานแทน เจ้าของ และระดับความเสี่ยง
- **Tools:** รายชื่อ [เซิร์ฟเวอร์ MCP](/th/admin/mcp-servers) ที่ลงทะเบียนไว้ทั้งหมด ได้แก่ ชื่อ โปรเจกต์ เจ้าของ ระดับความเสี่ยง สถานะการเปิดใช้งาน และคำอธิบายรายละเอียด
- **การกรองความเสี่ยง:** สามารถเลือกกรองตามระดับความเสี่ยง ได้แก่ สูง ปานกลาง หรือต่ำ เพื่อมุ่งเน้นไปยังส่วนที่สำคัญก่อน โดยระบบจะแสดงรายการที่มีความเสี่ยงสูงขึ้นมาเป็นอันดับแรก

## ความเสี่ยงและการระบุเจ้าของเครื่องมือ

เครื่องมือหรือเซิร์ฟเวอร์ MCP ในปัจจุบันจะเก็บข้อมูลเมทาดาตาสำหรับการกำกับดูแลในรูปแบบเดียวกันกับ agent ได้แก่ **ผู้รับผิดชอบ (owner)** **ระดับความเสี่ยง (risk tier)** และ**คำอธิบายรายละเอียด** ซึ่งกำหนดค่าได้ที่แท็บ [เซิร์ฟเวอร์ MCP](/th/admin/mcp-servers) ของโปรเจกต์เมื่อลงทะเบียนหรือแก้ไขเซิร์ฟเวอร์ ข้อมูลเหล่านี้จะช่วยให้แคตตาล็อกแสดงเจ้าของที่รับผิดชอบและระดับความเสี่ยงของแต่ละเครื่องมือได้ เช่นเดียวกับที่มีในระบบจัดการ agent

::: tip เน้นความโปร่งใสของข้อมูลก่อน ตามด้วยการบังคับใช้นโยบาย
แคตตาล็อกนี้ทำหน้าที่เป็นเพียงแดชบอร์ดภาพรวมการกำกับดูแลในรูปแบบสิทธิ์อ่านอย่างเดียว เพื่อแจ้งให้คุณทราบว่ามีสิ่งใดอยู่ในระบบบ้างและใครเป็นผู้รับผิดชอบ สำหรับการบังคับใช้สิทธิ์เข้าถึงจริง เช่น agent ตัวใดสามารถเรียกใช้งานเครื่องมือใดได้บ้าง จะทำงานอ้างอิงจากข้อมูลทะเบียนนี้และถือเป็นความสามารถในการประมวลผลที่แยกต่างหาก
:::
