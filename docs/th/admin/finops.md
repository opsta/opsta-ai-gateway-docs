# ระบบ FinOps การคิดค่าบริการคืนและการคาดการณ์งบประมาณ

หน้าจอ **FinOps** จะทำหน้าที่แปลงข้อมูลปริมาณการใช้งานบน gateway ให้เป็นข้อมูลตัวเลขทางการเงิน ได้แก่ **รายงานค่าใช้จ่าย (cost statement)** แยกตามแต่ละผู้เช่าในรอบเดือนที่เรียกเก็บเงินซึ่งสามารถส่งออกข้อมูลได้ และ **การคาดการณ์งบประมาณ (budget forecast)** ที่ช่วยประเมินแนวโน้มการใช้จ่ายจริงของเดือนนี้ ซึ่งมีความแตกต่างจากเครื่องมือตรวจจับค่าใช้จ่ายทั่วไป เนื่องจาก gateway จะบังคับใช้งบประมาณและจัดทำรายงานค่าใช้จ่ายโดยคำนวณจากชุดตัวเลขเดียวกัน ทำให้ข้อมูลการคิดเงินคืน (chargeback) ตรงกันกับข้อมูลที่ใช้จำกัดยอดค่าใช้จ่ายจริงเสมอ

::: info ผู้ที่มีสิทธิ์ในการดำเนินการนี้
**Org admin** (สิทธิ์เฉพาะในองค์กรของตนเอง) และ **Platform admin** โดยดำเนินการผ่านหน้าจอ **Usage → FinOps**
:::

## รายงานการคิดเงินคืน (Chargeback statement)

เมื่อเลือกเดือนที่ต้องการดูข้อมูล รายงานจะแจกแจงค่าใช้จ่ายแยกตาม **โปรเจกต์ → กลุ่ม → ผู้ใช้งาน → โมเดล** โดยจะคำนวณราคาที่ฝั่งเซิร์ฟเวอร์ด้วยอัตราค่าบริการชุดเดียวกับที่ระบบควบคุมงบประมาณใช้บังคับจริง แต่ละแถวจะแสดงจำนวน token ขาเข้า ขาออก และ token ที่ดึงมาจากแคช พร้อมระบุค่าใช้จ่ายจริงในสกุลเงิน **USD**

1. เปิดหน้า **Usage** เลือกองค์กรที่ต้องการ และเลื่อนลงไปยังส่วน **FinOps**
2. เลือก **เดือน (month)** ที่ต้องการตรวจสอบ (หากปล่อยว่างจะหมายถึงเดือนปัจจุบัน) จากนั้นคลิก **Refresh**
3. คลิก **Export CSV** เพื่อดาวน์โหลดรายงานสำหรับนำไปใช้ในระบบบัญชีการเงินของคุณ

![The FinOps view — chargeback statement + budget forecast](/images/finops.png)

::: tip ปริมาณการใช้งานจะถูกระบุผู้รับผิดชอบเสมอ และไม่มีการปล่อยให้ยอดเป็น 0 ดอลลาร์โดยไม่มีสาเหตุ
แต่ละแถวจะแสดงข้อมูลว่ามี**การประเมินราคาอย่างไร (pricedVia)** เช่น ประเมินตามราคาของโมเดลนั้น ๆ, ประเมินตาม**อัตราพื้นฐานเริ่มต้น (default fallback rate)** หรือหากคุณไม่ได้กำหนดค่าเริ่มต้นไว้ ระบบจะ**ทำเครื่องหมายว่าไม่ได้ระบุราคา (unpriced)** และแสดงในรายการ แทนการคิดเงินเป็น 0 ดอลลาร์โดยอัตโนมัติ ซึ่งคุณสามารถเข้ามากำหนดอัตราค่าบริการของโมเดลดังกล่าวได้ที่แท็บ [การกำหนดราคาโมเดล](/th/admin/pricing) จากนั้นจึงสั่งประมวลผลใหม่อีกครั้ง และไฟล์ CSV นี้มีความปลอดภัยในการเปิดใช้งานร่วมกับโปรแกรมตารางคำนวณทั่วไป เนื่องจากระบบได้ป้องกันการทำเขียนสูตรโจมตี (formula-injection) ไว้แล้ว
:::

## การคาดการณ์งบประมาณ (Budget forecast)

สำหรับงบประมาณ USD ทุกรายการที่คุณตั้งค่าไว้ ไม่ว่าจะเป็นในระดับโปรเจกต์หรือผู้ใช้งาน ระบบคาดการณ์จะแสดงข้อมูล **ยอดใช้จ่ายสะสมล่าสุด (spend-to-date)** **เปอร์เซ็นต์ของงบประมาณที่ใช้ไป (% of budget consumed)** และ **ยอดคาดการณ์เมื่อสิ้นเดือน (projected month-end)** ซึ่งคำนวณตามสัดส่วนการใช้งานตั้งแต่ต้นเดือนจนถึงปัจจุบัน ขอบเขตใดที่คาดว่าการใช้งานจะเกินงบประมาณจะถูกแสดงสัญลักษณ์แจ้งเตือน เพื่อให้คุณสามารถจัดการแก้ไขล่วงหน้าก่อนที่การใช้งานจริงจะถึงเพดาน แทนที่จะมารับทราบหลังจากที่เกตเวย์ปฏิเสธคำร้องขอไปแล้ว

คุณสามารถกำหนดค่างบประมาณได้ที่หน้า [งบประมาณและขีดจำกัด](/th/admin/budgets-and-limits) และหน้านี้จะทำหน้าที่แสดงรายงานเปรียบเทียบกับงบประมาณเหล่านั้น
